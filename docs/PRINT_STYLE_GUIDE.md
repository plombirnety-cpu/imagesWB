# PRINT_STYLE_GUIDE.md — типографика принтов v3

Рабочий стайлгайд для конвейера ANICLOT. Разбор трёх эталонных принтов владельца
(Тодзи/Jujutsu Kaisen дуотон, Фауст VIII/Shaman King фэшн-эдиториал, Резе-Дэндзи/
Chainsaw Man ризограф-поп) и правила, по которым `art_director.py` + новый модуль
типографики v3 (расширение `typography.py`) должны собирать текст НА принте так,
чтобы он выглядел как дизайнерский элемент, а не наклейка слогана.

Документ адресован разработчику, который это реализует — числа, доли, имена
файлов даны так, чтобы не было развилок "как решить самому".

---

## 0. Диагноз: почему v2 (under/punch/kana_side) не дотягивает до эталонов

v2 решила главную проблему v1 (текст оторван от фигуры), но у неё три системных
пробела относительно эталонов:

1. **Цвет текста — из фиксированной палитры `_TITLE_COLORS` (6 хардкод-цветов
   red/orange/white/yellow/purple/black), не из самой иллюстрации.** На эталонах
   текст всегда сделан ИЗ цветов конкретного арта (фиолетовый+бирюзовый Тодзи,
   зелёный+мятный Фауст, розовый+жёлтый Резе) — ни один из этих оттенков не
   входит в `_TITLE_COLORS/six`. Это единственная причина, по которой партия
   принтов выглядит "одним и тем же шрифтом с одной покраской" — палитра текста
   не привязана к арту.
2. **Одна шрифтовая роль на весь текст.** Anton/Archivo/Bangers дают ровно один
   характер на весь принт. На эталонах минимум 2 роли (кисть-цитата + капс-подвал;
   антиква-заголовок + катакана-подзаголовок; готика-подвал + катакана-колонка).
3. **Нет структурного подвала-этикетки и нет режима "цитата в кавычках".** Это
   отдельные графические блоки на всех трёх эталонах — их не заменить существующими
   under/punch.

v3 закрывает все три пробела, не ломая v1/v2 (обратная совместимость сохраняется,
см. раздел 6).

## 1. ПАЛИТРА: извлечение и назначение цвета тексту

### 1.1 Извлечение доминантных цветов из иллюстрации

Новая функция `extract_palette(rgba: Image.Image, n: int = 4) -> list[tuple[int,int,int]]`
в новом модуле (см. раздел 6 про расположение кода).

Алгоритм:

1. Взять только пиксели с `alpha > 20` (порог, не строго `> 0` — убирает мусорные
   полупрозрачные края антиалиасинга, которые k-means иначе тянет в серый).
2. Понизить разрешение перед кластеризацией: `rgba.resize((128, 128*H//W))` (или
   аналог с сохранением пропорций) — не гонять k-means по полноразмерному
   изображению, это дорого и не нужно для 3-4 кластеров.
3. k-means по RGB (3 канала), `k = 4`, `n_init = 4`, `random_state` фиксированный
   (например `42`) — детерминированность обязательна: одна и та же картинка
   должна всегда давать одну и ту же палитру (иначе повторный прогон конвейера
   на том же диекате даст другой текст — недопустимо для регрессии/отладки).
   Если `sklearn` недоступен в окружении — допустима ручная реализация
   Lloyd's k-means на numpy (10-15 строк, порог сходимости `1e-2` или 10 итераций).
4. Отсортировать кластеры по **занимаемой площади** (доле пикселей), не по яркости.
5. Отбросить кластеры, доля которых `< 4%` от общего числа пикселей — шумовые/
   краевые кластеры не должны попадать в палитру текста.
6. Вернуть от 3 до 4 цветов (обычно после отсева шага 5 остаётся 3-4). Если
   после отсева осталось `< 3`, снизить порог шага 5 до `2%` и пересчитать —
   не отдавать палитру короче 3 цветов (тексту нужен минимум основной + акцент
   + контровый/тёмный).

Результат — `PrintPalette`, отсортированный по убыванию площади:
```python
palette = extract_palette(figure_rgba, n=4)
# [(168, 64, 214), (52, 196, 178), (245, 245, 240), (18, 12, 26)]
#  ^ dominant        ^ accent        ^ light          ^ dark/shadow
```

### 1.2 Назначение ролей внутри палитры

Отсортированный список делится на роли программно, без ручного подбора:

- **`dominant`** — цвет с наибольшей площадью. Обычно это цвет кожи/одежды/фона-
  ауры — НЕ использовать напрямую под заливку текста (см. 1.3 про контраст),
  но участвует в чередовании слов и как база для обводки.
- **`accent`** — второй по площади цвет, максимально отличный по HUE от
  `dominant` (если второй в списке слишком близок по тону — взять третий).
  Критерий "слишком близок": разница HSV-Hue `< 25°` И разница светлоты
  `< 20%` одновременно. `accent` — второй цвет чередования слов/строк.
- **`light`** — самый светлый цвет палитры (по `L` в HSL, `L > 70%`); если
  такого нет в палитре — взять фиксированный `(245, 245, 240)` (тёплый
  белый, НЕ чистый `#FFFFFF` — резче на печати). Используется для заливки
  текста на тёмных областях фигуры и как цвет обводки-канта.
- **`dark`** — самый тёмный цвет палитры (`L < 20%`); если нет — фиксированный
  `(18, 14, 20)`. Используется ТОЛЬКО для тени/обводки, никогда для заливки
  основного текста (кроме подвала на светлом дизайне, см. 1.5).

### 1.3 Правило контраста (заливка текста vs. область фигуры под текстом)

Перед отрисовкой каждого текстового блока считать среднюю яркость (luminance,
`0.299R+0.587G+0.114B`) области фигуры/фона ПОД будущим bounding box'ом текста
(не всего холста — именно локальной зоны, как уже делает `_best_under_row_frac`
в v2 для позиционирования; для выбора цвета применить ту же выборку пикселей).

- Если локальная luminance области `> 140` (светло) → заливка текста = `dark`
  или самый тёмный из `dominant/accent`, обводка = `light`.
- Если `<= 140` (темно) → заливка текста = `light` или самый светлый из
  `dominant/accent`, обводка = `dark`.
- **Никогда** не заливать текст цветом, у которого разница luminance с локальным
  фоном `< 35` — если после выбора по правилу выше разница всё ещё мала (бывает
  на "плоских" палитрах), принудительно взять оставшийся крайний цвет палитры
  (либо `light`, либо `dark` фикс-значение из 1.2).

### 1.4 Чередование цветов слов в цитате (`quote_bottom`, раздел 3)

Обязательное правило по эталонам 1 и 3 (Тодзи, Резе): слова цитаты ЧЕРЕДУЮТСЯ
между `dominant` и `accent`, по словам, не по буквам:

```
"CURSED SORCERERS ARE SO FRAGILE."
   accent   dominant accent dominant accent   (чередование от слова к слову)
```

Реализация: индекс цвета = `i % 2`, где `i` — порядковый номер слова (0-based),
цвет `[accent, dominant][i % 2]`. Кавычки-скобки `" "` — цвета первого слова
(не отдельный элемент чередования). Если слов `< 3` — чередование всё равно
применяется (даже 2 слова красятся в разные цвета, это часть приёма, не
костыль для длинных строк).

### 1.5 Подвал-этикетка (`collection_footer`, раздел 3)

Три капс-блока в ряд: `BRAND | TITLE | CHARACTER`. Правило по эталону 1: цвета
чередуются ВНУТРИ каждого блока по словам (первое слово `dominant`, второе —
`accent`), а не блок целиком одним цветом. Если блок — одно слово (например
`ANICLOT`), красить целиком в `dominant` для первого блока, `accent` для
второго, `dominant` для третьего (чередование по номеру блока: `[dominant,
accent, dominant]`).

### 1.6 Обводка/тень — всегда из палитры, никогда чёрная "по умолчанию"

Функции `_hard_shadow_text`/`_draw_spaced` в `typography.py` принимают
`shadow_color`/`stroke_fill` параметром — в v3 туда ВСЕГДА передаётся `dark`
или `light` из `extract_palette`, никогда захардкоженная константа. Единственное
исключение — деградация без палитры (см. 6.3 fallback).

## 2. ШРИФТОВЫЕ РОЛИ

Максимум **2-3 роли на один принт** (см. таблицу комбинаций в разделе 3.6).
Все шрифты — Google Fonts, лицензия OFL (Open Font License), скачиваются как
статические `.ttf` (не variable, кроме уже имеющегося `NotoSansJP[wght].ttf`)
и кладутся в `fonts/`.

| Роль | Назначение | Файл (положить в `fonts/`) | Источник (Google Fonts) | Насечки/характер |
|---|---|---|---|---|
| `quote` | цитата в кавычках, кистевой/маркерный характер | `PermanentMarker-Regular.ttf` | Permanent Marker (Font Squirrel/GF, OFL) | рукописный маркер, толстый, читаемый в капсе и нижнем регистре |
| `quote_alt` (запасной вариант `quote`, более "кисть", менее "маркер") | цитата, если тема мрачнее/взрослее | `CaveatBrush-Regular.ttf` | Caveat Brush (Google Fonts, OFL) | широкая кисть, наклон, экспрессия |
| `display` | огромный высококонтрастный заголовок, фэшн-антиква | `PlayfairDisplay-Black.ttf` | Playfair Display (Google Fonts, OFL), вес Black/900 | высокий контраст штриха, тонкие засечки — Vogue-эффект |
| `gothic` | трэш/хоррор/поп-тема, готика/орнамент | `Cinzel-Black.ttf` (заголовочный вариант) ИЛИ `UnifrakturCook-Bold.ttf` для явной чёрной готики | Cinzel (Google Fonts, OFL) / UnifrakturCook (Google Fonts, OFL) | Cinzel — римская капитель с орнаментальным весом (универсальнее для мелкого подвала); UnifrakturCook — ломаный готический шрифт (только для акцентных крупных слов, нечитаем мелко) |
| `caps` (подвал, уже есть) | капс-гротеск структурного подвала | `Anton-Regular.ttf` / `ArchivoBlack-Regular.ttf` (уже в проекте) | Anton / Archivo Black (уже есть) | без изменений |
| `jp` (уже есть) | японская вертикальная типографика | `NotoSansJP[wght].ttf`, ось Weight=900 | Noto Sans JP (уже есть) | без изменений |

Правила выбора шрифтовой пары под настроение темы (используется в разделе 4):

- **Дуотон-цитатный** (мрачные/жёсткие персонажи, кровь/тени, как Тодзи): `quote`
  или `quote_alt` (кисть) для цитаты + `caps` (Anton) для подвала + `jp` для кандзи.
- **Фэшн-эдиториал** (элегантные/статусные персонажи, как Фауст VIII): `display`
  (Playfair Black) для заголовка + `jp` для подзаголовка-катаканы. Подвал в этом
  режиме НЕ используется (см. 3.5) — максимум `display` + `jp`, то есть 2 роли.
- **Поп-трэш/хоррор** (агрессивные/кислотные темы, как Резе/Дэндзи): `gothic`
  (Cinzel) для подвала + `quote` (Permanent Marker) ИЛИ прямое использование
  `gothic` для цитаты тоже (цитата и подвал одной готической ролью — так проще
  и ближе к эталону 3, где подвал и цитата визуально из одного шрифтового
  семейства) + `jp` для колонки катаканы.

**Жёсткое ограничение:** ни один принт не должен использовать больше 3 логических
ролей одновременно (роль `jp` считается отдельно от латинских ролей, но входит
в лимит 3). Комбинация `quote + display + gothic` в одном принте — запрещена.

## 3. КОМПОЗИЦИОННЫЕ РЕЖИМЫ v3

Новый набор `TEXT_MODES_V3`, расширяющий (не заменяющий) существующий
`TEXT_MODES = ("none", "under", "punch", "kana_side")`. v3 добавляет:

```python
TEXT_MODES_V3 = ("quote_bottom", "kanji_on", "editorial", "collection_footer")
```

Режимы v3 могут комбинироваться (это отличие от v2, где на входе всегда ровно
один режим) — на входе `compose_text_v3` принимает **список** режимов плюс флаг
`brand_label` (см. 3.5). Список применяется в порядке: `kanji_on` →
`quote_bottom` → `collection_footer` (порядок фиксированный, не зависит от
порядка в списке на входе — гарантирует одинаковую глубину наложения слоёв
на любом принте).

### 3.1 `quote_bottom` — цитата в кавычках под фигурой

- Позиция: центр по X относительно `alpha_bbox` фигуры; низ текстового блока —
  на `bottom_margin = 4%` высоты холста НИЖЕ нижнего края bbox фигуры (то есть
  ПОД фигурой, не внахлёст, в отличие от `punch`). Если после добавления цитаты
  общая высота содержимого превышает исходный холст — холст расширяется вниз
  (тот же приём, что `_compose_under`/`_compose_punch`: рисуем на увеличенном
  прозрачном канвасе, затем `_crop_to_content`).
- Кегль: одна строка цитаты (без кавычек) — старт `10%` ширины фигуры (`fig_w`),
  минимум `5%` (`_MIN_FONT_FRAC_QUOTE = 0.05`, ниже общего `_MIN_FONT_FRAC=0.07`,
  потому что кистевые шрифты визуально "тяжелее" при том же кегле). Если строка
  на минимуме всё ещё не влезает в `max_width = fig_w * 0.92` — переносится на
  вторую строку (баланс слов 55/45 по аналогии с `_split_slogan`), кегль второй
  строки не уменьшается относительно первой (обе строки — один размер, цитата
  не должна "лесенкой" мельчать).
- Кавычки `" "` — отдельные глифы того же шрифта `quote`, размер `1.15x` от
  кегля текста, цвет = цвет первого слова цитаты (см. 1.4).
- Обводка: `stroke_width = int(font_size * 0.04)`, цвет — `dark`/`light` по
  правилу контраста 1.3, единая для всей цитаты (обводка НЕ чередуется, только
  заливка слов).
- Межстрочный интервал (если 2 строки): `1.15x` высоты строки.
- Точка в конце цитаты — цвет последнего слова (продолжает чередование).

### 3.2 `kanji_on` — вертикальная колонка кандзи/катаканы имени НА фигуре

Отличие от уже имеющегося `kana_side` (колонка СБОКУ, у края фигуры,
перекрытие ~35%): `kanji_on` — колонка **внутри** силуэта, у центра масс фигуры
по X, перекрытие почти полное (глифы читаются НА теле/одежде персонажа как
деталь принта, не сбоку).

- Позиция по X: центр bbox фигуры (`fx0 + fig_w/2`), допустимое смещение
  `± 10% fig_w` в сторону, где под колонкой более ровный (менее контрастный по
  альфа-краю) участок силуэта — не обязательно вычислять автоматически на
  первой итерации, разрешён фиксированный центр `fx0 + fig_w * 0.5` как базовая
  реализация, автоподбор — опционально/потом.
- Позиция по Y: колонка занимает `55%` высоты фигуры (`fig_h * 0.55`), центр
  колонки на `fy0 + fig_h * 0.55` (чуть ниже геометрического центра — попадает
  в район груди/живота, как на эталоне 1, а не на лицо).
- Кегль на глиф: старт `fig_w * 0.16`, подгоняется вниз, если суммарная высота
  колонки (`glyph_h * len(kanji) * 1.05`) превышает `fig_h * 0.55` — пересчитать
  размер пропорционально (тот же приём, что уже есть в `_compose_kana_side`
  через `_glyph_dims`/`total_h`).
- Заливка: `light` (белый/светлый из палитры) с обводкой `dark`,
  `stroke_width = int(kegль * 0.09)` — толще, чем в `kana_side`
  (`* 0.025`), потому что здесь глиф лежит ПРЯМО на детализированной иллюстрации
  (не на относительно спокойном хромакей-поле сбоку) и должен читаться поверх
  контраста ткани/теней. По эталону 1 обводка явно плотная, буквы "вырезаны"
  из фигуры белым с тёмным кантом.
- Прозрачность заливки: `95%` (`alpha=242` из 255) — не 100%, лёгкая
  полупрозрачность даёт "впечатанность" в текстуру персонажа вместо аппликации
  поверх (заметно на эталоне 3, где катакана полупрозрачно-розовая, `alpha`
  там ниже — см. отдельный пункт 3.2а для explicit-прозрачного варианта).
- **Вариант 3.2а (полупрозрачная катакана, как эталон 3):** если тема
  поп-трэш/хоррор (см. раздел 4) — прозрачность заливки `55%` (`alpha=140`),
  БЕЗ обводки (обводка `stroke_width=0`) — колонка читается как призрачный
  оттиск, не как аппликация. Включается тем же вызовом `kanji_on` с параметром
  `ghost=True`.

### 3.3 `editorial` — display-заголовок, переплетённый с фигурой

Самый сложный режим, единственный, который используется СОЛО (без комбинаций,
см. 3.6) — заголовок и есть вся композиция текста на принте.

- Текст: `character_en` персонажа (например `"FAUST VIII"`) капслоком, шрифт
  `display` (Playfair Display Black).
- Кегль: старт `fig_w * 0.30` (огромный, крупнее любого другого режима) —
  заголовок по ширине примерно равен ширине фигуры, допуск `max_width = fig_w
  * 1.05` (заголовку разрешено быть чуть ШИРЕ фигуры, в отличие от остальных
  режимов, где текст умещается в `0.90-0.92 fig_w`).
- Позиция по Y: верхняя треть фигуры, базовая линия текста на `fy0 + fig_h *
  0.30` (на уровне груди/плеч — переплетение с головой и плечами, как на
  эталоне 2).
- **Переплетение по глубине (обязательный приём, не опция):** заголовок рисуется
  в ДВА прохода вместо одного:
  1. Первый проход — полный текст на слое ПОД фигурой (тот же приём, что
     `_compose_under`: текстовый слой первым, фигура наклеивается поверх).
  2. Второй проход — тот же текст, но **обрезанный по маске** верхней половины
     bounding box фигуры (маска = альфа-канал фигуры, инвертированная по X:
     берётся левая или правая половина глифов, определяется по тому, где у
     конкретного дизайна находится "воздух" сбоку от головы — практическая
     реализация: обрезать по вертикальной линии `fx0 + fig_w * 0.5`, левая
     половина текста остаётся ПОД фигурой из прохода 1, правая половина
     дополнительно копируется НАД фигурой отдельным alpha_composite после
     наклейки фигуры). Результат: часть букв визуально "за" персонажем, часть
     "перед" — как в эталоне 2, где буквы FAUST VIII частично скрыты за головой,
     частично лежат поверх плеча.
  - Если реализация двухпроходной маски избыточно сложна для первой итерации —
    ДОПУСТИМЫЙ упрощённый fallback: весь заголовок рисуется ПОД фигурой (один
    проход, как `_compose_under`), а 2-3 последние буквы слова (например "III"
    в "FAUST VIII") дублируются ПОВЕРХ фигуры на своей исходной позиции — это
    даёт минимальный, но узнаваемый эффект переплетения без полной глубинной
    маски. Пометить это в коде явным комментарием как v3.0-упрощение с TODO на
    полную маску в v3.1.
- Заливка: `dark` (чёрный/тёмный из палитры) — по эталону 2 заголовок именно
  тёмный/чёрный на светлой ауре, обводки НЕТ (антиква не нуждается в канте,
  жирный вес `Black` держит контраст сам).
- Подзаголовок (катакана тайтла, `title_jp` — транслитерация `title_en`, если
  есть, иначе не рисуется): шрифт `jp`, кегль `fig_w * 0.045`, позиция — под
  заголовком справа, `x = x_заголовка_право_край - text_w_подзаголовка`,
  `y = y_заголовка_baseline + line_height * 1.1`. Цвет — `dominant` или
  `accent` (не тот же `dark`, что заголовок — должен визуально отделяться
  как второстепенный элемент).
- Мини-логотип "Aniclot" антиквой над заголовком (по эталону 2): шрифт `display`,
  кегль `fig_w * 0.035`, позиция по центру, `y = y_заголовка_top - line_height
  * 1.3`, цвет `dominant`. Это единственное место, где бренд-слово рисуется
  ВНЕ `collection_footer` — допустимо только в режиме `editorial`, так как
  `editorial` не комбинируется с `collection_footer` (см. 3.6).

### 3.4 `collection_footer` — структурный подвал-этикетка

Три капс-блока в ряд под фигурой (и под цитатой, если есть `quote_bottom`):
`BRAND_LABEL | TITLE | CHARACTER`.

- Источник текста: `BRAND_LABEL` — конфиг-константа, по умолчанию `"ANICLOT
  COLLECTION"` (задаётся в конфиге проекта, не хардкод внутри функции
  типографики — см. раздел 6 про место константы); `TITLE` = `design["title_en"]`
  из art_director (может быть пустым — тогда блок пропускается, см. ниже);
  `CHARACTER` = `design["character_en"]` (может быть пустым — блок пропускается).
- Если оба `title_en` и `character_en` пусты — `collection_footer` рисует только
  один блок `BRAND_LABEL` по центру (не растягивать на всю ширину искусственно).
- Разделитель между блоками: символ `" | "` (пробел-пайп-пробел), цвет разделителя
  = `dark`/`light` по правилу контраста (не участвует в чередовании слов).
- Шрифт: `caps` (Anton, как уже есть) для дуотон-цитатного/эдиториал-фэшн
  сценария; `gothic` (Cinzel) для поп-трэш сценария (см. таблицу выбора в
  разделе 4). Капслок всегда.
- Кегль: `fig_w * 0.028` на латинские блоки (это заметно МЕЛЬЧЕ, чем в других
  режимах — подвал по эталонам всегда мелкий вспомогательный текст, не ударный
  элемент). Letter-spacing `int(fig_w * 0.004)`.
- Позиция: горизонтальная полоса по центру X фигуры, `y` = низ ВСЕГО содержимого
  принта (после `quote_bottom`, если есть) `+ отступ 3%` высоты холста. Если
  `collection_footer` — единственный текстовый элемент (без цитаты), `y` = `fy1
  + fig_h * 0.06` (сразу под фигурой, малый отступ).
- Чередование цвета слов ВНУТРИ блока — см. правило 1.5 (первое слово блока
  `dominant`, второе `accent`; для одиночных слов-блоков — чередование по
  номеру блока `[dominant, accent, dominant]`).
- Ограничение ширины: если сумма трёх блоков с разделителями превышает `fig_w *
  0.95` — уменьшить letter-spacing до `0`, затем кегль (не переносить подвал на
  две строки — по эталонам подвал всегда однострочный).

### 3.5 Таблица режимов v3 — сводка чисел

| Режим | Кегль (доля `fig_w`) | Мин. кегль | Позиция Y (доля `fig_h` от `fy0`, если не оговорено иначе) | Заливка | Обводка |
|---|---|---|---|---|---|
| `quote_bottom` | старт 0.10 | 0.05 | низ, `fy1 + холст*0.04` | чередование `accent/dominant` по словам | `dark`/`light` по контрасту, `stroke=0.04*size` |
| `kanji_on` | старт 0.16 | 0.08 | центр колонки на `0.55*fig_h` | `light`, alpha 242 (95%) | `dark`, `stroke=0.09*size` |
| `kanji_on(ghost=True)` | старт 0.16 | 0.08 | центр колонки на `0.55*fig_h` | `accent` или `light`, alpha 140 (55%) | нет |
| `editorial` заголовок | старт 0.30 | 0.14 | `0.30*fig_h` (baseline) | `dark` | нет |
| `editorial` подзаголовок jp | 0.045 | 0.03 | под заголовком, `+1.1 line_height` | `dominant`/`accent` | нет |
| `editorial` мини-лого | 0.035 | 0.02 | над заголовком, `-1.3 line_height` | `dominant` | нет |
| `collection_footer` | 0.028 | 0.018 | под всем содержимым, `+3%` холста | чередование `dominant/accent` по словам блока | `dark`/`light` по контрасту, `stroke=0.03*size` |

### 3.6 Правила комбинирования

- **`editorial` — всегда соло.** Не комбинируется ни с чем (заголовок уже
  занимает всю визуальную роль текста; добавление цитаты/подвала перегружает
  фэшн-композицию и разрушает "воздух" вокруг заголовка — насмотрено на
  эталоне 2, где кроме заголовка+подзаголовка+мини-лого больше нет текста).
- **`quote_bottom` + `kanji_on` + `collection_footer` — канонiчная тройка**
  (это ровно эталон 1: кандзи на фигуре + цитата снизу + подвал). Разрешена
  целиком или любым поднабором (например только `quote_bottom` +
  `collection_footer` без кандзи, если у персонажа нет японского имени —
  `kana` пустой в данных art_director).
- **`kanji_on(ghost=True)` + `collection_footer`** — типичная пара для
  поп-трэш/хоррор сценария (эталон 3: катакана-призрак сверху + подвал снизу),
  цитата в этом сценарии тоже используется, но идёт ВНУТРИ подвала как
  четвёртая строка по центру (см. 4.3), не отдельным `quote_bottom` блоком —
  иначе на кислотных перегруженных принтах текста становится больше 3 блоков
  и композиция "трещит".
- **Старые `under`/`punch`/`none` остаются в арсенале** как самостоятельные
  режимы v2 для дизайнов, где v3-эффекты избыточны (см. раздел 4, когда
  art_director должен выбирать v2 вместо v3).
- Максимум текстовых визуальных блоков на принте — **3**, независимо от того,
  сколько режимов скомбинировано (кандзи считается одним блоком, цитата —
  одним, подвал — одним, даже если внутри 3 капс-подблока).

## 4. Как арт-директор выбирает режим

### 4.1 Поля, которые Claude должен отдавать (расширение JSON-схемы art_director.py)

К существующим полям (`prompt, chroma, slogan, slogan_color, kana, character_en,
title_en, signature_props, text_mode`) добавить:

```
"text_modes_v3": "<пустой массив [] ИЛИ подмножество из
    ['quote_bottom','kanji_on','editorial','collection_footer'], согласно
    правилам комбинирования (editorial — всегда соло, максимум 3 визуальных
    блока). Пустой массив — используем text_mode (v2) вместо v3.>",
"quote": "<короткая цитата ЛАТИНИЦЕЙ в 4-10 слов, каноничная реплика
    персонажа или уместная авторская фраза под тему; ПУСТАЯ строка, если
    text_modes_v3 не включает quote_bottom>",
"name_jp": "<кандзи ИЛИ катакана имени персонажа для kanji_on/editorial-
    подзаголовка, 2-6 знаков; ПУСТАЯ строка, если неизвестно точное
    написание — приоритет иероглифов (кандзи) над катаканой, если у имени
    есть каноничное кандзи-написание (напр. 伏黒甚爾 для Тодзи Фусигуро),
    иначе катакана>",
"mood": "<одно слово: 'duotone_quote' | 'fashion_editorial' | 'pop_trash' —

    docs/PRINT_STYLE_GUIDE.md раздел 4>"
```

Поле `kana` (существующее) не удаляется — `name_jp` его расширяет (допускает
кандзи, не только катакану); санация `name_jp` в `_parse()` — тот же принцип,
что у `kana`, но допустить диапазон кандзи `一-鿿` в regex дополнительно
к катакане.

### 4.2 Признаки выбора `mood` (и следующего из него режима)

| Признак темы/промпта | `mood` | Режимы v3 | Шрифтовая пара |
|---|---|---|---|
| Мрачный/жёсткий персонаж, кровь/тени/проклятия, ограниченная 2-цветная аура (не радужная) | `duotone_quote` | `quote_bottom` + `kanji_on` + `collection_footer` | `quote`(Permanent Marker/Caveat Brush) + `caps`(Anton) + `jp` |
| Статусный/элегантный/"взрослый" персонаж, тема моды/аристократии/сцены, спокойная 2-тональная палитра (не кислотная) | `fashion_editorial` | `editorial` соло | `display`(Playfair Black) + `jp` (подзаголовок) |
| Агрессивный/кислотный/хоррор-комедийный персонаж, контрастные яркие брызги-эффекты, растровая/поп фактура в промпте | `pop_trash` | `kanji_on(ghost=True)` + `collection_footer` (+ цитата внутри подвала, см. 4.3) | `gothic`(Cinzel) + `jp` |
| Ничего из вышеперечисленного явно не подходит, простой/декоративный дизайн без явного персонажа-драмы | — (не назначать v3) | `text_modes_v3 = []`, использовать `text_mode` v2 (`under`/`punch`/`none`) | без изменений (v2 шрифты) |

Практическое правило для Claude (сформулировать в системном промпте
`art_director.py` близко к этому): **v3-режимы включать примерно в 55-65%
дизайнов** (не в 100% — иначе конвейер снова монотонный, теперь просто на
новом наборе приёмов вместо старого); v2 (`under`/`punch`/`none`) держать
для остальных 35-45%, отдавая предпочтение `none`/`under` для тем без сильного
персонажа-драмы (машины, бытовые концепты — `character_en` пустой).

### 4.3 Особый случай — цитата внутри подвала (`pop_trash`)

Когда `mood == pop_trash` и есть `quote`, но `quote_bottom` не включён отдельным
блоком (правило 3.6 — не больше 3 блоков) — цитата рисуется КАК ЧЕТВЁРТАЯ
строка структурного подвала, по центру, шрифтом `gothic` в кегле `fig_w *
0.032` (чуть крупнее двух строк капс-блоков `0.028`, но заметно мельче
самостоятельного `quote_bottom` 0.10) — это ровно позиция цитаты на эталоне 3
("по центру внизу цитата... номер коллекции крупно как графика" — цитата там
часть подвальной группы, не отдельно висящий блок).

## 5. АНТИПРАВИЛА

Явные запреты для реализации и код-ревью:

1. **Запрещено** заливать текст любым цветом из `_TITLE_COLORS` (шести
   хардкод-цветов v2), если активен любой режим v3 — только цвета из
   `extract_palette` конкретного диеката. `_TITLE_COLORS` остаётся только для
   старых v1/v2 стилей (обратная совместимость), не подмешивается в v3-путь
   ни при каком fallback, кроме случая полного отказа палитры (см. 6.3, и
   даже там — не `_TITLE_COLORS`, а нейтральный чёрно-белый дуотон).
2. **Запрещено** использовать одну шрифтовую роль на весь принт, если активны
   ≥2 текстовых блока v3 (например `quote_bottom` + `collection_footer` не
   могут оба рисоваться `caps`/Anton — цитата обязана быть `quote`-ролью,
   даже если технически можно было бы уместить обе Anton'ом).
3. **Запрещено** более 3 логических шрифтовых ролей на одном принте (раздел 2)
   и более 3 текстовых визуальных блоков (раздел 3.6).
4. **Запрещено** ужимать кегль текста ниже указанных `_MIN_FONT_FRAC`-порогов
   каждого режима (таблица 3.5) ради того, чтобы уместить длинную цитату/
   подвал в одну строку — вместо ужимания ниже минимума: переносить на
   дополнительную строку (цитата) или сокращать источник (Claude должен
   присылать `quote`/`title_en`/`character_en` в пределах разумной длины —
   `quote` ограничивается санацией в `_parse()` до, например, 70 символов,
   аналогично текущему `slogan[:34]`).
5. **Запрещено** располагать текст в отрыве от фигуры (пустая полоса снизу
   без взаимодействия с bbox фигуры) — это уже было запрещено в v2
   (`_crop_to_content`, позиционирование от `_alpha_bbox`) и остаётся
   обязательным для всех v3-режимов без исключений.
6. **Запрещено** рисовать обводку/тень чёрным цветом "по умолчанию", если
   `dark` из палитры — не чёрный (например на дуотон-фиолетовом принте тень
   должна быть тёмно-фиолетовой из палитры, не нейтрально-чёрной `#000000`).
7. **Запрещено** комбинировать `editorial` с `quote_bottom`/`kanji_on`/
   `collection_footer` (раздел 3.6) — при ошибочном получении такого набора
   от Claude код должен приоритизировать `editorial` и отбросить остальные
   режимы из списка (защитный код в `_parse`/сборщике, не падать, но и не
   рисовать всё сразу).
8. **Запрещено** одинаковое positioning/цвет чередования цитаты на разных
   принтах одной темы-серии — чередование ВСЕГДА начинается с `accent` для
   первого слова (фиксированное правило 1.4), но сама пара `accent/dominant`
   меняется от принта к принту вместе с палитрой — если два принта в батче
   выдают визуально одинаковый дуэт цветов, это сигнал, что `extract_palette`
   выродилась (баг, не стиль) — не маскировать вручную зафиксированной парой.

## 6. Технические заметки для реализации

### 6.1 Расположение кода

Рекомендуется новый файл `typography_v3.py` рядом с `typography.py` (не
переписывать `typography.py` целиком) с публичной функцией:

```python
def compose_text_v3(figure_rgba, modes: list[str], design: dict,
                     brand_label: str = "ANICLOT COLLECTION") -> Image.Image
```

`design` — тот же словарь, что возвращает `art_director.make_ideas` (расширенный
полями из 4.1). `typography_v3.py` импортирует низкоуровневые примитивы из
`typography.py` (`_font`, `_measure`, `_draw_spaced`, `_hard_shadow_text`,
`_alpha_bbox`, `_crop_to_content`, `_fit_lines_min_size`) — они уже достаточно
общие, дублировать их не нужно (в отличие от исторической ситуации с
`_TITLE_COLORS`, продублированной из `batch_print.py` — так делать в v3 не надо,
раз `typography.py` теперь можно импортировать напрямую внутри проекта).

### 6.2 Функция `extract_palette`

Расположить в `typography_v3.py` (или отдельном `palette.py`, если планируется
переиспользование вне типографики, например для подбора цвета мокап-фона —
на усмотрение разработчика, но не в `typography.py`, чтобы не тянуть `sklearn`
в модуль, от которого зависит текущий стабильный v1/v2-путь).

### 6.3 Fallback без палитры / без sklearn

Если извлечение палитры падает (пустая альфа, `sklearn` не установлен и ручной
k-means не сошёлся за разумное число итераций) — вернуть нейтральную
безопасную палитру `[(235,235,230), (35,30,40), (235,235,230), (20,18,24)]`
(тёплый белый/тёмно-фиолетовый дуотон) и залогировать warning — НЕ падать
конвейер целиком из-за одного диеката.

### 6.4 Порядок наложения слоёв (z-order) при комбинации режимов

Обязательный порядок отрисовки (снизу вверх), чтобы взаимное перекрытие текст/
фигура было идентичным на всех дизайнах:
1. `kanji_on` текстовый слой (под фигурой, как `_compose_under`).
2. Фигура (`alpha_composite` поверх слоя 1).
3. `kanji_on` "надфигурный" остаток, если реализована полная переплетающая
   маска (аналогично `editorial`, опционально для kanji_on — по эталону 1
   кандзи полностью НА фигуре, не нужно "за", так что для `kanji_on`
   допустим единственный проход ПОВЕРХ фигуры вместо двухпроходного —
   упрощение специально разрешено, в отличие от `editorial`, где переплетение
   обязательно).
   
   *Уточнение: для `kanji_on` (не `editorial`) правильный порядок —
   ОБРАТНЫЙ: сначала фигура, потом кандзи поверх неё (полупрозрачно/с
   обводкой). Двухпроходная маска нужна только `editorial`.*
3. `quote_bottom` слой (под fy1 фигуры, не пересекается с ней геометрически —
   порядок наложения неважен, но рисовать после фигуры для простоты кода).
4. `collection_footer` слой (ниже всего, рисуется последним).

### 6.5 Регресс-тест / калибровка

Аналогично существующему `preview_all` в `typography.py` — добавить
`preview_all_v3(rgba, design)`, прогоняющий все допустимые комбинации из 3.6
на одном диекате для визуальной калибровки перед подключением к батч-пайплайну.

## 7. Итоговая шпаргалка (для быстрой сверки при код-ревью)

- Палитра текста = `extract_palette(figure_rgba)`, никогда `_TITLE_COLORS` в v3.
- Максимум 3 шрифтовые роли, максимум 3 текстовых блока на принт.
- Цитата — чередование слов `accent/dominant`, кисть/маркер-шрифт, под фигурой.
- Кандзи/катакана — на фигуре (`kanji_on`) или сбоку (`kana_side` v2), японский
  шрифт всегда `NotoSansJP` вес 900.
- Подвал — три капс-блока, чередование по словам, всегда мелкий (`0.028 fig_w`).
- `editorial` — соло, антиква, буквы переплетены с фигурой по глубине.
- `mood` определяет и палитровые роли выбора, и шрифтовую пару, и набор режимов —
  одно решение Claude, три согласованных следствия в коде.
